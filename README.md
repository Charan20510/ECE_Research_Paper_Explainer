# ECE Research Paper Explainer

## Stage 1 – PDF Upload System

This repository implements the first stage of the project, a production‑quality
PDF ingestion pipeline using Django. The goal of Stage 1 is to allow users to
upload research papers in PDF format and have the system extract and store the
text as well as persist the raw content for downstream analysis.

### Features implemented so far

- **Django backend** with a `papers` app.
- **Model**: `UploadedPaper` stores the file, title, upload timestamp and the
  cleaned text.
- **File handling**: uploaded files are saved under `media/papers/`.
- **Extraction utils**:
  - `extract_text_from_pdf()` uses `pdfplumber` (with PyMuPDF fallback) to
    pull text from PDFs.
  - `clean_text_pipeline()` normalises whitespace and removes junk.
  - `save_extracted_content()` writes both `.txt` and `.json` copies into a
    structured `extracted_content/<paper-name>/` directory.
- **Views & forms**:
  - Rich upload page with drag‑and‑drop, file validation, and Bootstrap styling.
  - Results are shown inline; success/failure messages are provided.
- **Static assets**: custom CSS/JS under `static/uploader` implementing the
  interactive UI.
- **Dependency management**: requirements file lists all needed packages
  (Django, pdfplumber, etc.).
- **Robustness**: automatic migration generation, deferred imports to avoid
  startup failures, and folder naming collision handling.

### How to run

1. Create and activate a virtual environment.
2. Install dependencies:
   ```bash
   pip install -r requirements.txt
   ```
3. Apply migrations:
   ```bash
   python manage.py makemigrations
   python manage.py migrate
   ```
4. Start the server:
   ```bash
   python manage.py runserver
   ```
5. Open `http://127.0.0.1:8000/` and upload a PDF.

### Notes

- Stage 1 is modular and ready to integrate Stage 2 (section detection).
- All functionality uses Django best practices and clean architecture.
- The system runs on SQLite by default but can be pointed to PostgreSQL
  if needed.

## Stage 2 – Section Segmentation & Detection

This implementation built the Section Segmentation Engine. This algorithm takes the extracted full text from Stage 1, parses it chronologically using rule-based Natural Language Processing (regex), and sorts it securely into structured JSON maps. 

### Features implemented in Stage 2
- **Rule-Based Mapping Model**: `papers/segmentation.py` built to accurately classify unstructured text into logical divisions: `abstract`, `introduction`, `literature_review`, `methodology`, `analysis`, `results`, `conclusion`, and `references`.
- **Edge-Case Resilience**: The parser was thoroughly tested and handles Title case (`Related Work`), ALL CAPS (`III. METHODOLOGY`), Lowercase missing structure, implicit top-page abstracts, and trailing references blocks.
- **Automated Pipeline**: When a PDF is uploaded, `detect_sections()` executes entirely in the backend `views.py` without touching frontend UX.
- **Section Persistence**: Automatically generates and saves `sections.json` to the document repository under `extracted_content/<paper-name>/`.
- **Unit Testing Suite**: `papers/tests.py` includes robust end-to-end unit tests confirming classification accuracy.
